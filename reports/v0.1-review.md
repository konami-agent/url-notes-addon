# URL Notes Add-on v0.1 Review

## Implemented behavior

- Provides a Manifest V3 WebExtension popup for Firefox and Microsoft Edge.
- Uses local browser extension storage only; notes are not sent to a server.
- Shows the current active tab URL as a normalized note key in the popup.
- Stores one note per normalized URL key.
- URL key normalization lowercases the scheme and host, hash fragments are ignored, a trailing slash is trimmed when the path is not the root path, and query strings are preserved.
- Auto-saves note edits from the popup after a short debounce.
- Deletes the stored note when the user saves blank note text.
- Reloads a saved note when the same normalized URL is opened again.
- Keeps different normalized URLs as separate notes.
- Exports all stored notes as schema-versioned JSON with `schemaVersion: 1`.
- Imports schema-versioned JSON into local storage, normalizing imported URLs through the same note-key rules.
- Rejects invalid imports before saving imported entries, avoiding partial import writes.
- Includes validation, lint, build-zip, and GitHub Actions CI checks.
- Provides Firefox temporary add-on and Microsoft Edge load-unpacked instructions plus a manual smoke checklist.

## Limitations

- The UI is popup-only; there is no content-script sidebar, floating note, context menu, or keyboard shortcut.
- Storage is local browser storage only. There is no account, cloud sync, cross-device sync, or backup service.
- Query strings are preserved by design in v0.1, so tracking or cache-busting parameters can create separate notes for pages that a user may consider equivalent.
- There is no search, note list, or overview screen.
- Notes are plain text. There is no markdown preview, rich text, attachments, tagging, or pinning.
- Domain-level notes are not supported; notes are keyed to normalized full URLs.
- Import conflict handling is simple: imported notes overwrite existing notes for the same normalized key.
- Manual browser smoke testing is documented but not automated in Firefox or Edge.
- The v0.1 package is a local distributable zip only; it has not been published to Firefox Add-ons or Microsoft Edge Add-ons.

## Verification status

The current project includes automated coverage for URL normalization/storage behavior, browser API adapter behavior, popup interactions, scaffold invariants, extension validation, and zip packaging. The expected release checks are:

```bash
npm test
npm run lint
npm run validate:extension
npm run build:zip
```

The CI workflow runs the same core validation path on push.
