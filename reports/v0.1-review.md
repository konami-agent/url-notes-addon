# URL Notes Add-on v0.1 Review

## Implemented behavior

- Provides a Manifest V3 WebExtension popup for Firefox and Microsoft Edge.
- Uses local browser extension storage only; notes are not sent to a server.
- Shows the current active tab URL as a normalized note key in the popup.
- Stores one note per normalized URL key.
- URL key normalization lowercases the scheme and host, hash fragments are ignored, a trailing slash is trimmed when the path is not the root path, and query strings are preserved by default.
- Provides an opt-in local setting to ignore query strings for future note-key lookups when tracking/search parameters create duplicate URL notes.
- Rejects unsupported active tabs and unsafe active keys, including non-HTTP(S) URLs and credential-bearing web URLs.
- Domain notes store site-level reminders separately from URL notes for the current HTTP(S) host.
- Auto-saves URL-note and domain-note edits from the popup after a short debounce.
- Deletes the stored URL note or domain note when the user saves blank note text.
- Reloads saved notes when the same normalized URL or host is opened again.
- Keeps different normalized URLs as separate URL notes while sharing domain notes across the same host.
- Lists saved URL notes and domain notes in the popup, with local search over keys and note text.
- Renders a small local Markdown preview subset for URL notes and domain notes while treating raw HTML as text.
- Exports URL notes and domain notes as schema-versioned JSON with `schemaVersion: 1`.
- Imports schema-versioned JSON into local storage, normalizing imported URL and domain keys through the same note-key rules.
- Rejects invalid or conflicting imports before saving imported entries, avoiding partial import writes across URL-note and domain-note namespaces.
- Includes validation, lint, build-zip, local release artifact, and GitHub Actions CI checks.
- Provides Firefox temporary add-on and Microsoft Edge load-unpacked instructions, a manual smoke checklist, and a manual smoke evidence template for privacy-safe release review.

## Limitations

- The UI is popup-only; there is no content-script sidebar, floating note, context menu, or keyboard shortcut.
- Storage is local browser storage only. There is no account, cloud sync, cross-device sync, or backup service.
- Query strings are preserved by default in v0.1, so tracking or cache-busting parameters can create separate notes unless the user opts into ignoring query strings for future lookups.
- The ignore-query setting changes future lookups only; it does not migrate, merge, or deduplicate existing URL notes.
- Import conflict handling intentionally rejects conflicting duplicate keys within the same backup; it does not provide an interactive merge UI.
- Manual browser smoke testing is documented and templated but not automated in Firefox or Edge.
- The v0.1 package is a local distributable zip and GitHub Release asset only; it has not been published to Firefox Add-ons or Microsoft Edge Add-ons.

## Verification status

The current project includes automated coverage for URL normalization/storage behavior, domain-note behavior, import/export atomicity and validation, browser API adapter behavior, popup interactions, Markdown preview safety, scaffold invariants, extension validation, release artifact generation, and zip packaging. The expected release checks are:

```bash
npm test
npm run lint
npm run validate:extension
npm run build:zip
npm run build:release
```

The CI workflow runs the core validation path on push and can also be manually dispatched for verification recovery. Release builds publish a deterministic zip and `SHA256SUMS` checksum asset for reviewer-verifiable evidence.
